Overview

  • A week ago, you got promoted! You are now the CEO's secretary.
  • It's only been a week since you've been the secretary, and it's been tiring. One reason is the inefficient scheduling system of the company.
  • In this exercise, you will implement a program that automatically tells the boss what his task is at a specific day and time.

Instructions

  • In Python, create a schedule tracker program.
  • The program will take two inputs: the day and the time.
  • The day argument is an integer corresponding to a day of the week, starting from Sunday. Sundays are equivalent to a one (1), Mondays to a two (2), and so on until Saturday.
  • The time argument can be any of the following strings: ['morning', 'lunch', 'afternoon', 'evening'].
  • The following rules should be checked in order:
  • If the day is Sunday or Saturday, print 'weekend'
  • If time is lunch, print 'break'
  • If time is morning, print 'standup'
  • If time is evening, print 'clock-out'
  • If time is afternoon and it is Monday, Wednesday, or Friday, print 'client'
  • If time is afternoon and it is Tuesday or Thursday, print 'paperwork'

Test Cases

  • You must print the boss' schedule to the terminal exactly as the syntax states above.
  • You may assume that the dates argument will always be an integer from 1-7.
  • Your may assume that the time argument will always be the following: ['morning', 'lunch', 'afternoon', 'evening'].
