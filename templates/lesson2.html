Lesson 2: Conditionals, Loops

Indentation

  • If you have done the first set of exercises, you may have noticed that Syntax requires that you indent your code after the function.
  • In Python, indentation is an important concept. Python relies on indentation to identify the scope of your code.
  • Say you have a function. instead of using curly-brackets like many languages, Python identifies the code specific for that function with indents: 
                    
    def function():
        # These comments are indented the way you indent for functions!
        # If you do not indent your code, Python will not understand that it is for that specific function only!
  • Even outside of Python, programmers indent their code to be able to easily identify which code belongs to which parent function!
  • It is very good practice to indent your code to improve readability.

Conditions

  • Remember booleans? The True or False values? How about operators?
  • Python supports logical conditions from mathematics. Type a statement, and if it is true, you get a True value. If it isn't, you get False. 
                    
    print(5 > 10)   # Mathematics logic. Is 5 greater than 10?

    # Output
    False
  • These so-called 'conditions' can be used in several ways. Conditionals and loops are the most common use cases of conditions.

If Statements

  • A conditional is simply a sentence that expresses conditions, and the results if said conditions are met.
  • In programming, conditionals are fundamental building blocks that tell a computer to do certain tasks only if a condition is met.
  • One such conditional is an if statement. Basically an essential on every programming language, if statements are conditionals translated into code.
  • In Python, if statements function very much like English conditionals! 
                    
    # if statements use conditionals and require a colon!
    if [CONDITION IS TRUE]:
        [DO AN ACTION]
  • The syntax for if statements in Python are simple. Type a statement that can be evaluated as either True or False. If it is True, execute what is inside. If it isn't, the code block is skipped.
  • Say we want to print out if x is less than y. The if statement below evaluates the conditional. As the condition is True, an output gets printed! 
                    
    x = 8
    y = 14

    if x < y:
        print('x is less than y')

    # Output
    'x is less than y'
  • Boolean values and variables can be used directly inside of an if statement. As if statements use booleans to determine an action, the same logic applies! 
                    
    x = 8
    y = 14
    is_x_less_y = x < y
    
    if is_x_less_y:
        print('x is less than y')

    # Output
    'x is less than y'
  • But what if you want to do something if x is greater than y?

If-Else Statements

  • If a condition fails, the code inside the if statement gets skipped. If-else statements can be used to instruct a computer to do something when a condition fails.
  • Again, the syntax is much like English conditionals. If a condition is true, do this, ELSE, do that instead. 
                    
    x = 16
    y = 14

    if x < y:
        print('x is less than y')
    else:
        print('x is greater than y')

    # Output
    'x is greater than y'
  • In the code above, as the if statement failed, the else statement was instead executed. Since x was not less than y, the output printed out 'x is greater than y' isntead!
  • Else statements act like a last resort for when every if statement inside a so-called conditional chain.
  • Conditional chains are blocks of code that test for every possible condition given and executes the first block that returns True.
  • Else statements trigger if nothing in the chain was true. But conditional chains are built from three components, if statements, else statements, and else if statements.

Else-If Statements

  • If statements can be used consecutively. However, in most situations, multiple if statements are inefficient. They use up more lines of code, more keystrokes, more memory, and more time.
  • Take the code below for example. While, yes, the code is logically correct and gives the correct output, there is a much more efficient way of writing it. 
                    
    x = 2
    y = 2

    if x > y:
        print('x is greater than y')
    
    if x < y:
        print('x is less than y')
    
    if x = y:
        print('x is equal to y')

    # Output
    'x is equal to 2'
  • Else statements work inside conditional chains, but to solve the given problem above, we had to write three sepate if statements, and therefore, three conditional chains.
  • Else if statements (elif) allow you to write multiple if statements in one conditional chain. They are a combination of if and else statements.
  • Elif is Python's way of saying to check this condition instead if the above condition is false. Python reads code from top to bottom, remember? The same applies for conditional chains.
  • Take the code below, for example. Same problem as before, but this time with less keystrokes and a more elegant solution. 
                    
    x = 2
    y = 2

    if x > 2:
        print('x is greater than 2')
    elif x < 2:
        print('x is less than 2')
    else:
        print('x is equal to 2')

    # Output
    'x is equal to 2'
  • Combining if, else-if, and else statements can allow for comprehensive decision-making structures.

Logical Operators

  • One if statement can have multiple conditions inside. To do this, one can use logical operators (see lesson 1).
  • In the code below, the if statement only triggers if both conditions inside are True. As one wais false, the code block was skipped. 
                    
    x = 5
    y = 2

    if x < 4 AND y == 5:
        print('output!')
  • To get a refresher on logical operators, see lesson 1.

Nested If

  • In programming, nesting means to put one block of code inside another.
  • You have have if statements inside of other if statements. This is called a nested if.
  • In the given example below, the program called two print functions, one from the main if statement, and the other from the nested if. 
                    
    x = 20

    if x > 0:
        print('x is positive')
        if x % 2 == 0:
            print('x is even')
        else:
            print('x is odd')
    
    # Output
    'x is positive'
    'x is even'

Pass Statement

  • If statements cannot be empty. If for some reason, an if statement does not have any content, the pass statement can be used to avoid getting an error. It also exits out of the if statement, skipping code underneath. 
                    
    a = 10
    b = 20

    if a < b:
        pass
  • Pass statements are usually used as a placeholder for when code is in development. Pass statements can also be used inside other contexts, some we'll see later on.
  • Pass statements can also be used inside other contexts, some we'll see later on.

Loops

  • Repeating lines of code is a common practice in programming.
  • However, simply copying and pasting code is not only a waste of many resources but also simply does not work most of the time.
  • Loops are sequences of code that repeat until a certain condition is met.
  • Python has two loop commands, for loops and while loops.

For Loops

  • A for loop is used to iterate over a sequence (lists, dictionaries, sets, tuples, strings)
  • Don't worry! More in depth discussions on these data types will be found later on.
  • For loops will execute lines of code under it for each item inside of a sequence.
  • Say you have a string. A for loop will iterate over every character before stopping. 
                    
    string = bye

    for x in string:    # variables in a for loop are used for indexing—getting the current item in a sequence
        print('character')
  • The code above will print 'character' thrice as the variable (string) has three characters. Say a list or dictionary also has three elements. If you use a for loop, it will also repeat thrice.
  • The range() function can be used to loop through code a set number of times. It returns a sequence of numbers by default, starts at 0, and increments by 1 by default. 
                    
    for i in range(5):
        print('bark')
  • The code above prints bark 5 times, as specified by the number inside the range function.
  • It is also possible to change how the range function counts by passing multiple arguments. 
                    
    range(x)        # returns a sequence from 0 to x
    range(x, y)     # returns a sequence from x to y
    range(x, y, z)  # returns a sequence from x to y, incrementing by z

While Loops

  • A while loop executes code as long as a condition is true.
  • In this example, the code will print x as long as x is less than 5: 
                    
    x = 1
    while x < 5:
        print(x)
        x += 1
  • It is important to make sure that your while loop has a condition that can become false, or else it will continue forever. The code above increments x by 1 every loop, making sure it hits the conditional after a few loops.

Nested Loops

  • Like if statements and functions, loops can be nested on top of one another.
  • This can be used for solutions to problems that programmers often encounter. The code below prints the multiplication table from 1 to 3! 
                    
    for i in range(3):
        for j in range(3):
            print(i * j)

Pass, Continue, Break

  • As you already know, pass statements are used to implement code with no function. This can be useful in loops and as placeholders as loops cannot be empty. 
                    
    for i in range(10):
        pass    # TODO
  • Continue statements stop the current interation of a loop and continues with the next. In the example below, the code is skipped if x is not an even number. 
                    
    for x in range(10):
        if x % 2 != 0:
            continue
        print(f('{x} is an even number')
  • Break statements stop a loop altogether. This is specially useful for stopping a loop when a certain test-case is already met. 
                    
    x = 0
    
    for i in range(100):
        x += i
        if x > 100:
            break
